# MA Refactor and Cleanup Plan

## Purpose
This plan refactors MA into a cleaner, easier-to-maintain codebase by:
- removing dead or temporary scripts,
- extracting reusable logic into focused modules,
- improving script-level readability with human-friendly comments,
- cleaning up client HTML/CSS/JS structure and maintainability.

## Outcomes (Definition of Done)
- No dead scripts remain in runtime paths.
- Temporary/debug scripts are either removed or moved to a clearly non-runtime archive folder.
- Core files are broken into smaller, cohesive modules with clear responsibilities.
- All major runtime files include concise, high-value comments describing what/why/how.
- Frontend layout and styles are organized into predictable sections/components.
- Smoke tests pass for server startup, config, and key UI flows.
