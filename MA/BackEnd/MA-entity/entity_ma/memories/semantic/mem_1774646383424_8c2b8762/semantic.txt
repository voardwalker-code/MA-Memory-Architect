// ── Services · Pulse Engine ────────────────────────────────────────────────
//
// HOW BACKGROUND TASKS WORK:
// The pulse engine runs recurring tasks on timers, like a heartbeat
// that keeps things healthy.  Each "pulse" is a named timer that
// fires at regular intervals:
//
//   • Health pulse    — scans system health every few minutes
//   • Chore pulse     — processes items from the chores queue
//   • Custom pulses   — user-defined recurring tasks
//
// CHORES:
// Chores are stored in MA-Config/chores.json.  They're like a to-do
// list for the AI — things it should work on when it has free time.
// The pulse engine picks up chores and processes them automatically.
//
// CONFIGURATION:
// Pulse timing is configured in MA-Config/pulse-config.json.
// Each pulse has an interval (how often it fires) and can be
// individually started/stopped.
//
// WHAT USES THIS:
//   MA-Server-standalone.js — starts the pulse engine after boot
//   svc-slash-commands.js   — exposes /pulse and /chore commands
//   route-pulse.js          — HTTP endpoints for pulse management
//
// EXPORTS:
//   init, startAll, stopAll, startPulse, stopPulse,
//   getPulseStatus, getPulseConfig, savePulseConfig,
//   getChores, addChore, updateChore, removeChore,
//   readLog, LOG_DIR, CHORE_PATH, PULSE_CFG
// ─────────────────────────────────────────────────────────────────────────────
