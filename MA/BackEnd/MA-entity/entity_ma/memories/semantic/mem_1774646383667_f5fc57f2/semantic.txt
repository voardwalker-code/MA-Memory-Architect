// ── Services · Worklog ─────────────────────────────────────────────────────
//
// HOW SESSION CONTINUITY WORKS:
// The worklog is MA's "notebook" — a persistent markdown file that
// records what happened during each work session.  When MA starts a
// new conversation, it can read the worklog to remember:
//
//   • What tasks were recently completed
//   • What the current active task is
//   • What steps have been done vs. what's remaining
//
// This is stored as a simple markdown file at:
//   MA-workspace/MA-WORKLOG.md
//
// After each task completes, core-chat.js calls recordTask() to
// append a summary.  The worklog is also included in the system
// prompt (via getSummaryForPrompt) so the AI has context about
// recent work.
//
// WHAT USES THIS:
//   MA-core.js          — exposes worklog to other modules
//   core-chat.js        — records completed tasks
//   core-context.js     — includes worklog summary in system prompt
//   route-worklog.js    — HTTP endpoints for reading/writing worklog
//
// EXPORTS:
//   read()               → string (raw markdown)
//   getState()           → {activeTask, steps, summary}
//   write(content)       — overwrite the worklog
//   recordTask(result)   — append a task summary
//   setActiveTask(...)   — mark a task as in-progress
//   markStepDone(...)    — mark a step as completed
//   getSummaryForPrompt() → string for the system prompt
//   WORKLOG_PATH         — absolute path to the worklog file
// ─────────────────────────────────────────────────────────────────────────────
