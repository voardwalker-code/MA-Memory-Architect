// ── Services · Project Archive ─────────────────────────────────────────────
//
// HOW PROJECT ARCHIVES WORK:
// When MA works on a project, every step, thought, decision, code block,
// and agent dispatch is recorded as a "node" in the project archive.
// These nodes are connected by weighted "edges" forming a knowledge graph.
//
// Think of it like a detective's evidence board:
//   • Each sticky note is a NODE (a step, thought, or artifact)
//   • The strings between them are EDGES (connections/relationships)
//   • Thicker strings = stronger connections (higher weight)
//
// NEKOCORE OS COMPATIBLE:
// Nodes follow the NekoCore OS memory schema with:
//   • Weights and importance scores
//   • Decay over time (older nodes matter less)
//   • Topic tags extracted by RAKE + YAKE
//   • A full connection graph for the predictive memory system
//
// NODE TYPES:
//   step, code, response, thought, decision, agent_dispatch,
//   tool_result, user_input, error, knowledge
//
// WHAT USES THIS:
//   MA-core.js          — loads archive module at boot
//   core-chat.js        — records steps during task execution
//   route-projects.js   — HTTP endpoints for project management
//
// EXPORTS:
//   createProject, getProject, listProjects, closeProject,
//   resumeProject, recordNode, getNode, listNodes, addEdge,
//   getGraph, getNodeEdges, reinforceEdge, recordStep,
//   recordStepWithKnowledge, exportForPredictive,
//   lookupByTopic, getArchiveStats, NODE_TYPES, ...
// ─────────────────────────────────────────────────────────────────────────────
