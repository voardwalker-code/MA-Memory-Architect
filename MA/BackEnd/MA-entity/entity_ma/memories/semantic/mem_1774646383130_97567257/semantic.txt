// ── Services · Agent Catalog ───────────────────────────────────────────────
//
// HOW AGENT DELEGATION WORKS:
// MA can create "agents" — specialized AI assistants, each with their
// own identity, capabilities, and conversation history.  Think of them
// like hiring specialists for different jobs:
//
//   • A coding agent for writing code
//   • A research agent for gathering information
//   • A writing agent for documentation
//
// Each agent is a folder inside MA-entity/:
//   MA-entity/agent_{id}/
//     agent.json       — the agent's profile (name, role, capabilities)
//     prompt-history/  — log of every prompt sent to this agent
//
// This file provides CRUD operations (Create, Read, Update, Delete)
// for the agent catalog, plus search functions to find agents by
// role or capability.
//
// WHAT USES THIS:
//   MA-core.js          — loads the agent catalog during boot
//   seed-agents.js      — creates default agents on first run
//   route-system.js     — exposes agent info via the API
//
// EXPORTS:
//   AGENT_SCHEMA, PROMPT_ENTRY_SCHEMA, createAgent, getAgent,
//   updateAgent, deleteAgent, listAgents, findAgentsByRole,
//   findAgentsByCapability, getCatalogSummary, recordPrompt,
//   getPromptHistory, searchPromptHistory, validateAgent, ENTITY_DIR
// ─────────────────────────────────────────────────────────────────────────────
