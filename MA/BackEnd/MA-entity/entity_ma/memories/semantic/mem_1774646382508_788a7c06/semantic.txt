// ── Infra · Command Executor ───────────────────────────────────────────────
//
// HOW SAFE COMMAND EXECUTION WORKS:
// Sometimes the AI needs to run a shell command (like "npm install"
// or "git status").  But letting an AI run ANY command would be
// dangerous — it could delete files, install malware, or worse.
//
// This executor acts like a security guard:
//   1. WHITELIST CHECK — only pre-approved commands are allowed
//      (configured in MA-Config/cmd-whitelist.json)
//   2. WORKSPACE JAIL — commands run in the workspace directory,
//      not the system root
//   3. NO SHELL MODE — uses spawn() with shell=false, preventing
//      shell injection attacks (no &&, ||, ;, pipes, etc.)
//   4. OUTPUT LIMIT — captures max 16 KB of output per stream
//   5. TIMEOUT — kills commands that run longer than 60 seconds
//
// SANITIZATION:
// The command parser splits the input into [command, ...args] and
// validates the command name against the whitelist.  Arguments are
// NOT passed through a shell interpreter.
//
// WHAT USES THIS:
//   MA-Server-standalone.js — passes execCommand to route handlers
//   svc-slash-commands.js   — /cmd slash command
//   tools-web.js            — cmd_run tool
//   MA-cli.js               — CLI mode command execution
//
// EXPORTS:
//   parseCommand, execCommand, getAvailableCommands,
//   getWhitelist, whitelistAdd, whitelistRemove, whitelistReset
// ─────────────────────────────────────────────────────────────────────────────
