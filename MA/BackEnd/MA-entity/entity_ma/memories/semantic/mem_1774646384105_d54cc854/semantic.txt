## 2. The Comment Voice

Write comments like you're explaining to a smart 10-year-old.

### DO — Use analogies

```javascript
// Think of it like a phone operator — when someone calls "ws_read",
// the operator connects them to the file-reading department.
```

```javascript
// Think of it like a detective's evidence board:
//   • Each sticky note is a NODE (a step, thought, or artifact)
//   • The strings between them are EDGES (connections/relationships)
//   • Thicker strings = stronger connections (higher weight)
```

### DO — Explain the "why", not just the "what"

```javascript
// BAD:
// Normalize the path
const safe = path.normalize(p);

// GOOD:
// Normalize the path to prevent directory traversal attacks.
// Without this, someone could pass "../../etc/passwd" and escape
// the workspace sandbox.
const safe = path.normalize(p);
```

### DO — Use plain words

```javascript
// BAD:
// Implements the Okapi BM25 probabilistic relevance framework

// GOOD:
// Scores how well a document matches a search query (like Google ranking)
```

### DON'T — State the obvious

```javascript
// BAD:
const fs = require('fs'); // require the fs module
let count = 0;            // set count to zero

// GOOD:
const fs = require('fs');
let count = 0;
```
