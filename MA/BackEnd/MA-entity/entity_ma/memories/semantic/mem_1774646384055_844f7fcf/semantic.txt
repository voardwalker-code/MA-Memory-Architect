## The Dependency Audit Technique

Before moving ANY file, build a complete dependency map:

```bash
# Find every require() in the project
grep -rn "require(" --include="*.js" .

# For a specific file, find who imports it
grep -rn "require.*filename" --include="*.js" .

# For a specific file, find what it imports
grep -n "require(" filename.js
```

**Why this matters:** If you move a file without updating its importers,
the server crashes. The dependency map tells you exactly what to update.

## The safeRequire() Pattern

Every orchestrator file uses this pattern:

```javascript
function safeRequire(modulePath, label) {
  try {
    return require(modulePath);
  } catch (err) {
    console.warn(`⚠ Could not load "${label}": ${err.message}`);
    return null;
  }
}
```

**Why:** If one module is broken (syntax error, missing dependency),
the rest of the system still works. The broken module logs a warning
instead of crashing everything.

**Where to use it:** In orchestrator/dispatcher files that load many
sub-modules. NOT in leaf modules (those should fail loudly).
